# Project Titan Design Book

## 06 Component Library

## 1. この章の目的

この章では、Project Titan で使う共通UIコンポーネントを定義する。

目的は、同じ役割のUIを、どの画面でも同じ見た目・同じ動きにすることである。

例えば、開始ボタン、カード、セット行、コピーアイコン、モーダルなどが画面ごとにバラバラだと、アプリ全体の完成度が下がる。

そのため、Project Titan では、以下のような部品を共通化する。

```text
ボタン
カード
セット行
コピーUI
入力欄
種目タブ
下タブ
モーダル
下部シート
トースト
グラフカード
設定行
```

---

## 2. Primary Button

### 2.1 役割

最も重要な操作に使う。

使う場所：

```text
トレーニング開始
記録する
保存して終了
ログイン
バックアップ実行
```

---

### 2.2 見た目

```text
高さ：48px
角丸：16px
背景：accentColor
文字：#F9FAFB
文字サイズ：16px
文字太さ：700
横幅：基本100%
```

---

### 2.3 例

```text
[トレーニング開始]
[保存して終了]
[記録する]
```

---

### 2.4 ルール

1画面にPrimary Buttonは基本1つまで。

理由は、重要な操作を迷わせないため。

## 6. Set Row

### 6.1 役割

1セット分の重量・回数を表示する。

Project Titanの最重要コンポーネントの1つ。

---

### 6.2 通常表示

```text
1　55.0kg　8回　✓
```

仕様：

```text
高さ：44〜52px
背景：#111C2D
角丸：12px
padding：0 12px
文字サイズ：14〜16px
数字：やや太め
```

---

### 6.3 未入力状態

```text
1　--kg　--回
```

未入力は `#9CA3AF` で表示する。

---

### 6.4 完了状態

```text
1　55.0kg　8回　✓
```

チェックは成功色。

```text
#22C55E
```

---

### 6.5 編集状態

セット行をタップすると、編集状態になる。

```text
背景：#162033
境界線：rgba(255,255,255,0.12)
```

---

## 7. Set Edit Panel

### 7.1 役割

セット行をタップしたときに表示する編集パネル。

重量・回数・コピー・微調整をまとめる。

---

### 7.2 表示内容

```text
Set 3

重量  57.5 kg   ⧉
回数  6 回       ⧉
メモ             ⧉

[-2.5] [+2.5]
[-1]   [+1]

詳細 ▼
[完了]
```

---

### 7.3 ルール

- 重量と回数を最優先で見せる
- コピーアイコンは補助
- メモとRPEは詳細に隠す
- パネルは画面下から出すのが理想

---

## 8. Copy Button / Copy Icon

### 8.1 役割

前回記録から必要な項目だけコピーする。

---

### 8.2 コピーできるもの

```text
重量
回数
メモ
セット全体
種目全体
メニュー全体
```

---

### 8.3 見た目

通常時は目立たせない。

```text
アイコン：⧉
サイズ：16〜18px
色：#9CA3AF
押下時：accentColor
```

---

### 8.4 表示ルール

通常表示では出しすぎない。

```text
通常表示：
1　55.0kg　8回　✓
```

編集時だけ表示する。

```text
重量 55.0kg ⧉
回数 8回 ⧉
メモ ⧉
```

---

### 8.5 フィードバック

コピー後は軽く反応。

```text
コピーしました
```

または、アイコンを一瞬チェックに変える。

長い通知は不要。

---

## 20. Empty State

---


---

---

## 21. Loading State

### 21.1 役割

同期や保存中に表示する。

---

### 21.2 例

```text

同期中...

```

---

### 21.3 ルール

長すぎるローディングは避ける。

保存はできるだけ即時反映し、裏で同期する。

---

## 22. Error State

### 22.1 役割

エラー時に原因と次の行動を伝える。

---

### 22.2 例

```text

通信状態を確認してください
[再試行]
```

---

### 22.3 ルール

エラー文は短くする。

原因と次の行動をセットで表示する。

---

## 23. この章の結論

Project Titan では、UI部品を必ず共通化する。

特に重要なコンポーネントは以下。

```text
Set Row
Set Edit Panel
Copy Button
Exercise Tab
Bottom Sheet
Primary Button
Card
Toast
```

このComponent Libraryにより、Codexが実装しても画面ごとにデザインがバラバラになりにくくなる。

次の章では **07_Codex_Implementation_Guide.md** として、実装時にCodexへ守らせるルールを作る。
