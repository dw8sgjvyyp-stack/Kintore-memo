# Project Titan Design Book

## 00 Project Titan Design Book

## 1. この資料の目的

この資料は、筋トレ記録アプリ **Kintore-memo / Project Titan v2** を開発するための総合設計書である。

Project Titan v2 は、今まで作ってきた `Kintore-memo` を土台にした大幅改良版である。

目的は、単なる多機能アプリを作ることではない。

最大の目的は、**トレーニングを止めない筋トレ記録アプリを作ること**である。

筋トレ中に集中すべきなのは、スマホ操作ではなく、以下である。

```text
次のセット
フォーム
重量
回数
休憩
体の感覚
```

そのため、このアプリでは、記録にかかる思考・入力・画面移動を最小化する。

---

## 2. Project Titan の中心思想

Project Titan の中心思想は以下。

```text
トレーニングに合わせて、アプリが変わる。
```

多くの筋トレアプリは、以下のような設計になっている。

```text
先にメニューを決める
↓
その通りに記録する
```

しかし実際のジムでは、以下のようなことが普通に起こる。

```text
マシンが空いていない
種目を変えたい
セット数を増やしたい
今日は軽めにしたい
予定にない種目を追加したい
```

だから Project Titan では、予定通りに進めることよりも、以下を重視する。

```text
その場で変えられること
```

---

## 3. 最重要方針

Project Titan で最も大事な方針は以下。

```text
1. 記録を止めない
2. 入力を少なくする
3. 戻る操作を減らす
4. 予定変更に対応する
5. 前回コピーを強制しない
6. セット数を固定しない
7. 使うほど自分専用になる
```

特に重要なのは、記録画面。

ホームや分析がきれいでも、記録画面が使いにくければ、このアプリの価値は下がる。

## 5. 各章の役割

### 01_Vision.md

Project Titan の目的・思想・ターゲット・成功条件をまとめる章。

このアプリが何を目指すのかを定義する。

重要な考え方：

```text
多機能な筋トレアプリではなく、
トレーニングを止めない筋トレ記録アプリ。
```

---

### 02_UX_Flow.md

ユーザーがアプリを開いてから、トレーニングを終えるまでの流れを決める章。

主に以下を定義する。

```text

ジムモード
セット記録
途中変更
終了サマリー
体重入力
睡眠入力
履歴確認
```

## 6. Codexが最初にやること

Codexに最初から実装させてはいけない。

最初は、以下を依頼する。

```text
docsフォルダ内のProject Titan Design Bookを読んでください。

まずは実装せず、現在のアプリ構成を確認し、
以下を報告してください。

1. 現在の主要ファイル構成
2. 記録画面を担当しているファイル
3. メニュー管理を担当しているファイル
4. データ保存を担当しているファイル
5. Firebase / PWA / Vercel設定に関係するファイル
6. Project Titan v2を実装する時に壊してはいけない部分

まだコード変更はしないでください。
```

---

## 7. Codexに守らせること

Codexは必ず以下を守る。

```text
3セット固定にしない
前回コピーを強制しない
記録中に戻る操作を強制しない
数字キーボードを使う
画面を縦に長くしすぎない
UI部品を共通化する
既存のVercel / Firebase / PWA設定を壊さない
既存データをなるべく壊さない
```

---

## 8. 実装の優先順位

最初から全部作らない。

順番は以下。

```text
v0.1 現状確認
v0.2 記録画面の土台改良
v0.3 前回記録・コピー機能
v0.4 ジムモード完成
v0.5 記録中の変更機能
v0.6 ホーム・メニュー管理
v0.7 体重・睡眠
v0.8 履歴・カレンダー
v0.9 分析
v1.0 個人用完成版
v2.0 公開候補版
```

---

## 9. 絶対に避けること

以下は避ける。

```text
最初に分析画面を作る
最初にログインだけ作る
最初に全画面を作り直す
デザインだけ先に凝る
Firebase構造を先に大幅変更する
App Store公開準備を先にする
```

理由は、記録画面が完成していないと、アプリの価値が決まらないため。

---

## 10. 成功条件

Project Titan v2 の成功条件は、機能の多さではない。

成功条件は、以下である。

```text
ジムで実際に使ったとき、記録が邪魔にならないこと。
```

具体的には、以下。

```text
1セット3秒以内で記録できる
セット数を自由に変えられる
重量・回数・メモを個別にコピーできる
前回コピーを使わずにも始められる
記録中に種目変更できる
記録中にメニュー変更できる
数字キーボードが出る
スクロールが少ない
片手で操作しやすい
毎日使っても疲れない
```

## 11. この資料の使い方

今後の流れは以下。

```text
1. このチャットで設計を考える
2. docsに保存する
3. Codexにdocsを読ませる
4. 小さく実装させる
5. Vercelで確認する
6. iPhoneで実機確認する
7. 問題点を整理する
8. docsを更新する
9. 再度Codexに修正させる
```

この繰り返しで、Project Titan v2を完成させる。

---

## 12. この章の結論

Project Titan Design Book は、Kintore-memo を Project Titan v2 に作り替えるための基準である。

この設計書の中心は、以下である。

```text
トレーニングを止めないこと
入力を少なくすること
途中変更に強くすること
記録画面を最優先すること
```

Codexは、このDesign Bookを基準に実装する。

思いつきで機能を追加せず、記録画面から段階的に作る。
