# Project Titan Design Book

## 05 Design System

## 1. この章の目的

この章では、Project Titan の見た目に関する共通ルールを定義する。

目的は、どの画面を見ても同じアプリに見えることである。

画面ごとに、以下がバラバラになると、アプリ全体の完成度が下がる。

- 色
- 余白
- 文字サイズ
- ボタン
- カード
- 入力欄
- タブ
- アイコン
- アニメーション

そのため、Project Titan では、すべての画面をこのDesign Systemに従って作る。

---

## 2. デザインの方向性

Project Titan のデザイン方向性は以下。

```text
ダークネイビー
高級感
集中感
コンパクト
片手操作
Apple風
筋トレ中に見やすい
```

派手すぎるデザインにはしない。

筋トレ中に必要なのは、装飾ではなく、**見やすさ・速さ・押しやすさ**である。

## 4. アクセントカラー

### 4.1 基本方針

背景・カード・文字色は固定。

変更できるのは、**アクセントカラーのみ**。

理由は、全体の統一感を守りながら、自分専用感を出すため。

---

### 4.2 デフォルト

```text
Red：#EF4444
```

赤は、筋トレアプリとして以下を表現しやすい。

```text
力強さ
集中
エネルギー
開始
記録
```

---

### 4.3 選べるアクセントカラー

```text
Red：#EF4444
Blue：#3B82F6
Purple：#8B5CF6
Green：#22C55E
Orange：#F59E0B
```

---

### 4.4 アクセントカラーを使う場所

```text
開始ボタン
記録ボタン
現在選択中のタブ
＋ボタン
アクティブ状態
重要な強調
グラフのメイン線
```

---

### 4.5 使いすぎ禁止

アクセントカラーを使いすぎると、どこが重要か分からなくなる。

赤や青などのアクセントは、画面内で多くても1〜3箇所に抑える。

---

### 5.2 注意

```text

```


---

### 5.3 危険・削除

```text

```

例：

```text

```

## 6. タイポグラフィ

### 6.1 基本フォント

iOS / Apple風を優先する。

```text
font-family:
- system-ui
- -apple-system
- BlinkMacSystemFont
- "SF Pro Display"
- "Helvetica Neue"
- sans-serif
```

---

### 6.2 文字サイズ

```text
大見出し：24px
画面タイトル：20px
カードタイトル：16〜18px
本文：14〜16px
補足：12〜13px
数字メイン：22〜28px
小さいラベル：11〜12px
```

---

### 6.3 数字の見せ方

重量・回数・時間などの数字は、視認性を高くする。

```text
font-weight: 700
letter-spacing: -0.02em
```

例：

```text
55.0kg
8回
01:30
```

ただし、大きすぎない。

記録画面では、数字が主役だが、画面を圧迫しないサイズにする。

---

### 6.4 文字の優先順位

#### 最重要

```text
現在の種目
重量
回数
タイマー
```

#### 重要

```text
メニュー名
セット番号
前回記録
次の操作
```

#### 補足

```text
日付
前回比
説明文
単位
```

## 7. 余白ルール

### 7.1 基本余白

```text
画面左右：16px
画面上部：12〜20px
カード内：14〜16px
カード間：10〜12px
ボタン間：8〜10px
セクション間：18〜24px
```

---

### 7.2 記録中画面の余白

ジムモードでは、余白を広く取りすぎない。

理由は、1画面内に以下を収めたいからである。

```text
現在種目
前回記録
今回セット
操作ボタン
タイマー
```

記録中画面では、以下を目安にする。

```text
カード内：12〜14px
カード間：8〜10px
```

---

### 7.3 設定・分析画面の余白

設定や分析は、記録中ほど急がない。

少し余裕を持たせる。

```text
カード内：16px
カード間：12〜14px
セクション間：24px
```

## 11. 入力欄

### 11.1 基本入力欄

```text
height: 44〜48px
background: #090F18
border: 1px solid rgba(255,255,255,0.08)
border-radius: 12px
color: #F9FAFB
font-size: 16px
```

---

### 11.2 数字入力

重量・回数・体重は、数字入力を最優先する。

```text
重量：inputmode="decimal"
回数：inputmode="numeric"
体重：inputmode="decimal"
RPE：inputmode="numeric"
```

日本語キーボードは禁止。

---

### 11.3 未入力表示

未入力の場合は、以下のように表示する。

```text
--kg
--回
```

色はサブ文字。

```text
#9CA3AF
```

## 12. セット行

### 12.1 通常表示

```text
1　55.0kg　8回　✓
```

仕様：

```text
height: 44〜52px
background: #111C2D
border-radius: 12px
padding: 0 12px
```

---

### 12.2 編集中

編集中のセットは強調する。

```text
background: #162033
border: 1px solid rgba(255,255,255,0.12)
```

---

### 12.3 コピーアイコン

通常時は出しすぎない。

編集時に、以下のように表示する。

```text
重量 55.0kg ⧉
回数 8回 ⧉
メモ ⧉
```

コピーアイコンは補助であり、主役にしない。

---

## 13. タブ

### 13.1 下タブ

アプリ全体の主要タブ。

候補：

```text
ホーム
記録
履歴
分析
設定
```

仕様：

```text
height: 56〜64px
background: rgba(9,15,24,0.92)
border-top: 1px solid rgba(255,255,255,0.08)
```

---

### 13.2 種目タブ

記録中に現在の種目を切り替える。

```text
ベンチ
フライ
肩
＋追加
```

仕様：

```text
height: 34〜38px
border-radius: 999px
active: accentColor
inactive: #111C2D
```

横スクロールは許可するが、画面全体が横にはみ出すのは禁止。

---

## 14. アイコン

### 14.1 基本方針

アイコンはシンプルにする。

iOSらしさを出すなら、SF Symbols風の線アイコンを使う。

---

### 14.2 使うアイコン例

```text
＋：追加
✓：完了
…：詳細
⧉：コピー
🗑：削除
⌕：検索
▼：展開
```

絵文字をそのまま多用しすぎない。

実装時は統一されたアイコンライブラリを使うのが望ましい。

---

## 15. アニメーション

### 15.1 基本方針

アニメーションは短く、自然にする。

派手にしない。

---

### 15.2 推奨時間

```text
ボタン反応：80〜120ms
カード展開：160〜220ms
画面遷移：220〜280ms
モーダル表示：180〜240ms
保存完了：100〜160ms
```

---

### 15.3 使う場所

```text
セット完了
コピー完了
カード展開
メニュー変更
種目変更
下部シート表示
```

---

## 16. フィードバック

### 16.1 コピー完了

コピー後は軽く反応を出す。

```text
コピーしました
```

または、小さくチェック表示。

長いトーストは不要。

---

### 16.2 保存完了

セット保存時は、以下を表示し、休憩タイマーを開始する。

```text
✓
```

---

### 16.3 危険操作

削除などは確認を出す。

ただし、確認を多くしすぎない。

---

## 17. レスポンシブ

### 17.1 iPhone優先

まずはiPhone幅を最優先。

```text
375px〜430px
```

---

### 17.2 iPad / PC

iPadやPCでは、中央にスマホ幅のコンテナを置く。

```text
max-width: 430px
margin: 0 auto
```

横に広げすぎない。

理由は、筋トレ中に使うアプリなので、スマホUIを基準にするため。

---

## 18. この章の結論

Project Titan のデザインシステムは、**ダークネイビーを土台にした、コンパクトで高級感のある筋トレ記録UI**である。

重要なのは以下。

```text
背景・カード・文字は固定
アクセントカラーだけ変更可能
記録中はコンパクト
数字は見やすく
ボタンは押しやすく
コピー機能は補助的に表示
余白は広すぎず狭すぎず
アニメーションは短く自然に
```

このDesign Systemに従うことで、どの画面を見ても統一感のあるアプリになる。
