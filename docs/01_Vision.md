# Project Titan Design Book

## 01 Vision Book

## 1. アプリ名

### 仮名

**筋トレMEMO Pro**

正式名称は後で決める。現時点では、アプリの中身・UX・設計思想を優先する。

---

## 2. プロジェクトの目的

このアプリの目的は、ただ筋トレを記録することではない。

本当の目的は、**筋トレ中の記録ストレスを限りなくゼロにすること**である。

筋トレ中に集中すべきなのは、次のセット、フォーム、重量、回数、体の感覚、休憩時間、今日のコンディションであって、スマホ操作ではない。

そのため、このアプリは「多機能な筋トレアプリ」ではなく、**トレーニングを止めない筋トレ記録アプリ**として設計する。

---

## 3. ミッション

### Mission

**筋トレ記録にかかる思考・入力・画面移動を最小化し、トレーニングに集中できる環境を作る。**

このアプリでは、ユーザーにできるだけ考えさせない。

理想は、以下の流れである。

```text
前回コピー
↓
必要な部分だけ修正
↓
自動保存
↓
次セットへ
```

---

## 4. ビジョン

### Vision

**日本で一番、実際のジムで使いやすい筋トレ記録アプリを作る。**

このアプリは、机の上で考えた理想的なトレーニングではなく、実際のジムで起こる状況を前提にする。

実際のジムでは、以下のようなことが普通に起こる。

- ベンチが埋まっている
- マシンが使えない
- 気分で種目を変えたい
- 肩や膝に違和感がある
- 今日はセット数を増やしたい
- 今日は少なめで終わりたい
- 予定にない種目を追加したい
- 一部の種目をスキップしたい
- 前回と同じ内容をすぐ入力したい

だから、このアプリは予定通りに進めることだけを前提にしない。

---

## 5. コアコンセプト

### Core Concept

**トレーニングに合わせて、アプリが変わる。**

多くの筋トレアプリは、以下のような設計になっている。

```text
先にメニューを決める
↓
その通りに記録する
```

しかし、実際のトレーニングでは予定通りにいかない。

だから Project Titan では、**予定通りやる**よりも、**その場で変えられる**ことを重視する。

---

## 6. Design Principles

Project Titan の設計原則は5つ。

### Principle 1：記録を止めない

筋トレ中に、長い入力や複雑な操作をさせない。

記録画面では、以下を徹底する。

- 入力項目を減らす
- キーボード入力を減らす
- 前回コピーを基本にする
- 自動保存する
- 次セットへ自動で進む
- 不要な確認画面を出しすぎない

目標は、**1セット3秒以内で記録できること**。

### Principle 2：戻らない

記録中に画面を戻らせない。

筋トレ中に、以下のような操作をさせない。

```text
戻る
↓
メニュー編集
↓
保存
↓
戻る
↓
記録再開
```

記録画面内で、以下を完結できるようにする。

- メニュー変更
- 種目変更
- 種目追加
- 種目スキップ
- セット追加
- セット削除
- 順番変更
- 前回コピー
- タイマー操作

例：

```text
Push ▼
↓
Pull / Legs / 自作メニューへ変更
```

```text
ベンチプレス ▼
↓
チェストプレスへ変更
```

戻る操作を減らすことで、トレーニングの流れを止めない。

### Principle 3：考えさせない

画面を見た瞬間に、次に何をすればいいか分かるUIにする。

記録中の画面では、以下だけを優先して表示する。

- 今のメニュー
- 今の種目
- セット数
- 重量
- 回数
- 前回記録
- 次に押すボタン

不要なものは出さない。

特に記録中は、以下を表示しない。

- 大きすぎるグラフ
- 長い説明文
- 大量の設定項目
- 使わない分析情報
- 大きすぎるカード

### Principle 4：変更を歓迎する

このアプリは、予定変更を前提にする。

ジムでは予定通りに進まないことが普通なので、以下の操作を簡単にできるようにする。

- 今日だけ種目を追加
- 今日だけ種目をスキップ
- 今日だけ順番を変える
- 今後もメニューに追加
- 今後もメニューから削除
- 記録中にメニュー変更
- 記録中に種目変更
- 記録中にセット数変更

特に重要なのは、**今日だけ**と**今後も**を分けること。

これにより、柔軟性と安全性を両立する。

### Principle 5：使うほど自分専用になる

使えば使うほど、自分のトレーニングに最適化される。

例：

- 最近使った種目
- お気に入り種目
- よく追加する種目
- よく使う重量
- 前回記録コピー
- 自作メニュー
- スマートテンプレート提案

最終的には、アプリを開いた瞬間に、**今日やること**がすぐ分かる状態を目指す。

## 7. ターゲットユーザー

### メインターゲット：筋トレを継続している中級者

特徴：

- 週2〜6回トレーニングする
- 前回の重量・回数を見たい
- セット数を自由に変えたい
- メニューを自作したい
- 種目をその場で変更したい
- 入力が面倒だと記録をやめる
- 分析も見たいが、記録中は邪魔されたくない

このアプリの中心ユーザーは、この層である。

### サブターゲット1：初心者

初心者に必要なのは、複雑な分析ではない。

必要なのは、以下である。

- 今日やるメニュー
- 簡単な入力
- 前回コピー
- 分かりやすい種目選択
- 達成感
- 迷わないUI

初心者には「全部できる」よりも、「迷わずできる」ことが重要。

### サブターゲット2：上級者

上級者に必要なのは、以下である。

- セット数自由変更
- 種目ごとの履歴
- 1RM推定
- ボリューム分析
- 部位別分析
- CSV / JSONエクスポート
- 細かいメモ
- テンプレート編集
- データの自由度

ただし、これらを記録画面に出しすぎない。

詳細分析や高度な設定は、記録画面ではなく別画面に分ける。

## 10. メニュー管理の方針

メニュー管理はシンプルにする。

初期テンプレートは、以下のみにする。

```text
Push
Pull
Legs
```

それ以外は、自作メニューとしてユーザーが自由に作る。

### 初期状態

表示するテンプレート：

```text
Push
Pull
Legs
自作メニュー
```

### 表示しない固定テンプレート

以下は初期テンプレートとして固定表示しない。

```text
Upper
Lower
Shoulder
Arms
Core
```

理由：

- 初期表示が多すぎると選びにくい
- 使わないテンプレートが邪魔になる
- 自作で作れる方が柔軟
- メニュー画面をシンプルにしたい

### テンプレート削除

Push / Pull / Legs も含めて、テンプレートは削除可能にする。

ただし、削除時は確認を出す。

例：

```text
Pushを削除しますか？

この操作は取り消せます。
履歴データは削除されません。

キャンセル / 削除
```

重要：

- テンプレート削除 ≠ 過去の履歴削除
- 履歴データは残す
- 削除したテンプレートは復元できるようにするのが理想

## 11. 記録中のメニュー変更

記録中でも、その場でメニューを変更できるようにする。

例：

```text
Push ▼
```

を押すと、以下が出る。

```text
Push
Pull
Legs
自作メニュー
```

そこから選ぶだけで、記録中のメニューを変更できる。

### 重要ルール

- 画面を戻らない
- 入力中の記録を消さない
- 変更前の記録を保持する
- 変更後にセット内容を引き継ぐか選べる
- 必要なら「今日だけ / 今後も」を選べる

---

## 12. 記録中の種目変更

記録中でも、その場で種目を変更できるようにする。

例：

```text
ベンチプレス ▼
```

を押すと、以下が出る。

```text
ベンチプレス
インクラインベンチ
ダンベルプレス
チェストプレス
種目を追加
```

### 種目変更時の選択

種目変更時には、必要に応じて確認を出す。

```text
現在のセット内容をどうしますか？

○ 引き継ぐ
○ リセットする
```

例：

ベンチプレスで入力中：

```text
55kg × 8
55kg × 8
55kg × 7
```

チェストプレスへ変更するときに、セット内容を引き継ぐかリセットするかを選択する。

---

## 13. クイック追加

記録中に、すぐ種目を追加できるようにする。

例：Push中にディップスを追加したい場合。

```text
＋種目追加
↓
ディップス
↓
追加
↓
そのまま記録開始
```

戻る必要はない。

### 追加後の扱い

トレーニング終了時、または設定に応じて、以下を選べる。

```text
今日だけ追加
今後もPushへ追加
お気に入り登録
```

## 14. クイックスキップ

今日はやらない種目を簡単にスキップできるようにする。

例：サイドレイズを長押し、またはメニューから操作する。

```text
今日だけスキップ
テンプレートから削除
```

を選択できる。

## 15. 今日だけ / 今後も

このアプリの重要機能。

予定変更をしたときに、**今日だけ**なのか、**今後も反映**なのかを選べるようにする。

### 例1：種目追加

今日はディップスを追加した。

選択肢：

```text
今日だけ追加
Pushに今後も追加
お気に入り登録
```

### 例2：種目スキップ

今日はサイドレイズをやらなかった。

選択肢：

```text
今日だけスキップ
Pushから削除
```

### 例3：順番変更

今日はショルダープレスを先にやった。

選択肢：

```text
今日だけこの順番
今後もこの順番にする
```

---

## 16. 終了時の「今日だけ / 今後も」確認設定

「今日だけ / 今後も」の確認は強制しない。

便利な機能だが、毎回表示されると記録速度を下げる可能性があるため、設定で選べるようにする。

### 設定項目

```text
設定
↓
トレーニング記録
↓
終了時の変更確認
```

選択肢は3つ。

### 1. 毎回確認する

すべての変更について、終了時に確認する。

確認対象：

- 種目追加
- 種目スキップ
- 種目変更
- メニュー変更
- 順番変更
- テンプレート変更

向いている人：

- 初心者
- テンプレートを慎重に管理したい人
- 変更をあとから整理したい人

### 2. 大きな変更だけ確認する

デフォルト設定。

大きな変更だけ、終了時に確認する。

確認するもの：

- 新しい種目を追加した
- 種目をテンプレートから削除した
- メニュー構成を大きく変えた
- 自作メニューに反映できそうな変更があった

確認しないもの：

- 重量を変えた
- 回数を変えた
- セット数を増やした
- セット数を減らした
- 今日だけ順番を変えた
- メモを書いた

向いている人：

- ほとんどのユーザー
- 入力速度と整理のバランスを取りたい人

### 3. 確認しない

終了時に確認を出さない。

すべての変更は基本的に、**今日だけ**として保存する。

テンプレートには自動反映しない。

向いている人：

- 記録速度を最優先したい人
- 毎回確認されるのが面倒な人
- トレーニング中の変更をテンプレートに反映したくない人

### デフォルト

**大きな変更だけ確認する**をデフォルトにする。

理由：

- 毎回確認は面倒
- 確認なしだと便利な提案を逃す
- 大きな変更だけなら邪魔になりにくい
- 「記録を止めない」という原則に合っている

## 18. 最初に完成させる画面

最初に作るべき画面は、**トレーニング記録画面**。

理由は、このアプリの価値の中心がここだから。

ホームや分析が綺麗でも、記録画面が面倒なら使われない。

### 最初の開発優先順位

```text
1. トレーニング記録画面
2. セット入力UI
3. 前回コピー
4. セット追加・削除
5. 記録中のメニュー変更
6. 記録中の種目変更
7. 今日だけ / 今後も
8. 終了時確認設定
```

## 19. 成功条件

このアプリの成功条件は、機能数ではない。

成功条件は、**ジムで実際に使ったとき、記録が邪魔にならないこと**である。

具体的には、以下を満たすこと。

- 1セット3秒以内で記録できる
- 数字キーボードだけで入力できる
- 前回コピーで入力を減らせる
- セット数を自由に変えられる
- 記録中に戻る操作がほぼない
- メニュー変更がその場でできる
- 種目変更がその場でできる
- 今日だけ追加ができる
- 今後も追加ができる
- 終了時確認を設定で選べる
- 片手で操作できる
- スクロールが少ない
- 毎日使っても疲れない

---

## 20. この章の結論

Project Titan は、**多機能な筋トレアプリ**ではなく、**トレーニングを止めない筋トレ記録アプリ**として作る。

最大の価値は、以下である。

- 記録の速さ
- 柔軟な変更
- 入力の少なさ
- 使うほど自分専用になること

このアプリは、予定通りに進む理想のジムではなく、予定が変わる現実のジムに合わせて設計する。
