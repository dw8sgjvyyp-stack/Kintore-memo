# Project Titan Design Book

## 08 Roadmap

## 1. この章の目的

この章では、Project Titan v2 の開発順番を決める。

目的は、以下を守ることである。

```text
思いつきで機能追加しない
一気に全部作らない
壊さずに段階的に改良する
```

Project Titan は、今ある `Kintore-memo` を土台にした大幅改良版である。

そのため、既存の公開環境・PWA・Firebase・データ保存を壊さずに進める。

---

## 2. 開発方針

### 基本方針

```text
設計
↓
小さく実装
↓
Vercelで確認
↓
iPhoneで実機確認
↓
問題点を整理
↓
docsを更新
↓
再実装
```

この流れを繰り返す。

---

## 3. 最優先は記録画面

Project Titan の中心は、記録画面。

ホームや分析が綺麗でも、記録画面が面倒なら使われない。

そのため、最初に作るのは以下である。

```text
ジムモード / 記録中画面
セット入力
コピー機能
セット追加・削除
数字キーボード
```

---

## 4. Version Plan

### v0.1：現状確認

目的：

```text
今のアプリ構造を把握する
壊してはいけないファイルを確認する
どこから改良するか決める
```

やること：

```text
現在のファイル構成確認
記録画面の担当ファイル確認
メニュー管理の担当ファイル確認
データ保存の担当ファイル確認
Firebase / PWA / Vercel設定確認
```

やらないこと：

```text
コード変更
UI変更
全画面作り直し
```

---

### v0.2：記録画面の土台改良

目的：

```text
記録画面をProject Titan仕様に近づける
```

やること：

```text
セット行をコンパクト化
3セット固定を廃止
＋セット追加
セット削除
重量・回数の数字キーボード
前回コピーを強制しないUI
```

成功条件：

```text
3セット固定ではない
セット追加できる
セット削除できる
数字キーボードが出る
画面が見やすい
```

---

### v0.3：前回記録・コピー機能

目的：

```text
入力の手間を減らす
```

やること：

```text
前回記録表示
前回をコピー
空で開始
前回のセット数だけコピー
重量だけコピー
回数だけコピー
メモだけコピー
セット全体コピー
種目全体コピー
```

成功条件：

```text
前回コピーが便利
でも強制されない
重量・回数・メモを個別にコピーできる
```

---

### v0.4：ジムモード完成

目的：

```text
トレーニング中に戻らず操作できる状態にする
```

やること：

```text
種目タブ
現在の種目表示
休憩タイマー
＋種目追加
次の種目
トレーニング終了
Bottom Sheetで編集
Toastでコピー完了表示
```

成功条件：

```text
記録中に画面移動が少ない
片手で操作しやすい
スクロールが少ない
```

---

### v0.5：記録中の変更機能

目的：

```text
現実のジムで予定変更に対応できるようにする
```

やること：

```text
記録中のメニュー変更
記録中の種目変更
種目追加
種目スキップ
順番変更
今日だけ / 今後も
終了時の変更確認
```

成功条件：

```text
メニュー変更しても記録が消えない
種目変更しても記録が消えない
今日だけ / 今後もを選べる
確認の出し方を設定できる
```

---

### v0.6：ホーム・メニュー管理

目的：

```text
トレーニング開始までを速くする
```

やること：

```text
ホーム整理
今日のおすすめメニューON/OFF
Push / Pull / Legs
自作メニュー
テンプレート削除
メニュー編集
メニュー複製
並び替え
```

成功条件：

```text
ホームがごちゃごちゃしていない
すぐトレーニングを始められる
自作メニューを選びやすい
```

---

### v0.7：体重・睡眠

目的：

```text
筋トレ記録とは別に、体重と睡眠を素早く記録できるようにする
```

やること：

```text
体重入力
体重履歴
体重グラフ
睡眠入力
睡眠履歴
睡眠時間自動計算
```

重要ルール：

```text
体重と睡眠は別々に管理する
同じ画面に詰め込みすぎない
```

---

### v0.8：履歴・カレンダー

目的：

```text
過去の記録を見返しやすくする
```

やること：

```text
履歴一覧
履歴詳細
カレンダー表示
日付別記録
種目別履歴
メモ表示
```

成功条件：

```text
いつ何をやったか分かる
過去の重量・回数が見やすい
メモも見返せる
```

---

### v0.9：分析

目的：

```text
成長を確認できるようにする
```

やること：

```text
重量推移
総重量
セット数
レップ数
部位割合
頻度
1RM推定
PR表示
体重推移
睡眠推移
```

重要ルール：

```text
分析は記録中には出さない
記録画面を重くしない
```

---

### v1.0：個人用完成版

目的：

```text
自分が毎日使える完成度にする
```

条件：

```text
記録画面が使いやすい
セット追加・削除が自然
コピー機能が便利
メニュー変更ができる
体重・睡眠が記録できる
履歴が見返せる
iPhoneで問題なく使える
PWAとしてホーム画面に追加できる
```

この段階では、まず **自分専用アプリとして完成** を目指す。

## 6. 実装してはいけない順番

以下は避ける。

```text
最初に分析画面を作る
最初にログインだけ作る
最初に全部の画面を作り直す
デザインだけ先に凝る
Firebase構造を先に大幅変更する
App Store公開準備を先にする
```

理由：

```text
記録画面が完成していないと、アプリの価値が決まらないため。
```

---

## 7. 毎回のレビュー基準

実装後は、以下の順番で確認する。

```text
1. 起動するか
2. 記録できるか
3. 入力が速いか
4. セット追加・削除できるか
5. 前回コピーを強制していないか
6. 数字キーボードが出るか
7. スクロールが多すぎないか
8. 見た目が統一されているか
9. iPhoneで使いやすいか
10. 既存機能が壊れていないか
```

---

## 8. Codexへの最初の実行順

最初にCodexへ渡す順番はこれ。

### Step 1：調査のみ

```text
docsを読んで、現在のアプリ構成を調査してください。
まだコード変更はしないでください。
```

### Step 2：記録画面v0.2

```text
記録画面だけをProject Titan仕様に近づけてください。
```

### Step 3：コピー機能

```text
重量・回数・メモの個別コピーを実装してください。
```

### Step 4：ジムモード

```text
記録中に戻らず操作できるUIへ改良してください。
```

---

## 9. この章の結論

Project Titan は、一気に全部作るのではなく、段階的に作る。

最初に作るべきなのは以下である。

```text
記録画面
セット入力
コピー機能
セット追加・削除
数字キーボード
```

v1.0 の目標は、まず **自分が毎日使える個人用完成版**。

その後、v2.0 で他人にも使える公開候補版を目指す。
